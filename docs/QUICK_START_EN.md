# Disher.io v2.6 Quick Start Guide

> **Versión en español:** [QUICK_START.md](./QUICK_START.md)

This guide will get you a functional restaurant platform in under 5 minutes.

---

## Prerequisites

- A server or device with at least 1GB of RAM.
- **Docker (v24+)** and **Docker Compose (v2.x)** pre-installed.
- Git to clone the repository.
- Port **80** available on your system. Port **443** is also required if you use a public domain (Let's Encrypt requires it for automatic HTTPS).
- If you use a cloud provider (Google Cloud, AWS, Azure...), ports must also be **opened in the provider's firewall** (see [Cloud Providers](#cloud-providers-firewall) section).

---

## Step 1 — Automated and Secure Installation

The automated installer is the only recommended method. It handles everything from configuration and security to service deployment.

```bash
# 1. Clone the repository
git clone https://github.com/ismailhaddouche/disherio.git
cd disherio

# 2. Grant execution permissions to the installer
chmod +x install.sh

# 3. Run the installer with superuser privileges
sudo ./install.sh
```

The script will automatically perform the following actions:
1.  **System Verification**: Checks that Docker, Docker Compose, and minimum RAM are available.
2.  **Security Configuration**: Generates secrets (`JWT_SECRET`) and secure database credentials.
3.  **Access Mode Selection**: The script will ask how you want to access the platform. Depending on your infrastructure, you'll need to choose one of these four variants:

### 1.1 Network Configuration Variants

During installation, the script will adapt the proxy behavior and SSL security based on the environment you choose. Understand the options before running the installer:

- **Public Domain (Online):** 
  - *Example:* `app.myrestaurant.com`
  - *Use:* For installations on cloud servers (VPS). Allows customers to access the application from anywhere in the world.
  - *Requirements:* A real domain pointing to your server's IP. Caddy will automatically provision an SSL certificate from **Let's Encrypt** (HTTPS). At the end of installation, the necessary **DNS records** (A and CAA) will be displayed.
- **Local Domain (mDNS / LAN):** 
  - *Example:* `disher.local`
  - *Use:* For installations physically located in your venue (e.g., a cash register or mini-PC) without depending on external internet connectivity.
  - *Requirements:* All devices (customer smartphones and waiters) must connect to the same venue WiFi network. No SSL certificates are generated.
- **Public IP (Online):** 
  - *Example:* `203.0.113.50`
  - *Use:* Installation on a cloud provider or public VPS, but without a contracted domain. Access using raw numbers.
  - *Requirements:* No SSL generated (travels over plain HTTP). Everyone must use the IP to access.
- **Local IP (LAN):** 
  - *Example:* `192.168.1.100`
  - *Use:* Local on-premise installation at your restaurant using the IP assigned by your router.
  - *Requirements:* Depends exclusively on the restaurant's WiFi network. It's essential to configure this IP as **static** in your router so QR codes don't stop working if the router restarts. No SSL is generated.

## Step 2 — First Login

At the end of installation, the script will display a summary with your initial credentials. **Save them in a secure place.**

If you chose a **public domain**, you'll also see a table with the DNS records you need to configure in your domain registrar (A and CAA records for Let's Encrypt). You can consult this information at any time by running:

```bash
sudo ./show-dns.sh
```

- **Access URL:** The one you selected during installation (e.g., `http://disher.local` or `https://your-domain.com`).
- **Admin Credentials:**
  - **Username:** `admin`
  - **Password:** The secure password generated by the installer.

Simply access the URL and log in with these credentials.

---

## Step 3 — Configure Your Restaurant

Once inside the admin panel, you can start customizing your restaurant:

1.  **Configure VAT (Required):** Go to `/admin/config` → **Billing** section and set the VAT percentage for your country (e.g., 21% in Spain, 16% in Mexico). **Without this step, the system cannot generate receipts or process payments correctly.**
2.  **Create Your Menu:** Go to `/admin/menu` to add categories, dishes, variants, and extras. Prices are entered with VAT included.
3.  **Define Your Tables:** In `/admin/config` → `Tables`, generate unique QR codes for each table.
4.  **Create Staff Accounts:** In `/admin/users`, add more users with `kitchen` and `cashier` roles.
5.  **Open Workstations:**
    - **Kitchen View (KDS):** `.../admin/kds`
    - **Point of Sale Terminal (POS):** `.../admin/pos`

## Cloud Providers — Firewall

When installed on a VPS or cloud instance, the provider has its **own network firewall**, independent of the operating system. Even if the application is running correctly, external access will be blocked until the necessary ports are opened.

- **Port 80** (HTTP) — mandatory for all installations.
- **Port 443** (HTTPS) — mandatory if you use a public domain with Let's Encrypt.

### Google Cloud (Compute Engine)

The most reliable way is from the **web console**, as network permissions are often limited from within the VM.

**Option A — Web console:**

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Navigate to **VPC Network → Firewall**
3. Click **"+ CREATE FIREWALL RULE"** and use these values:

| Field | Value |
|-------|-------|
| Name | `allow-http-80` |
| Network | `default` |
| Direction of traffic | `Ingress` |
| Action on match | `Allow` |
| Targets | `All instances in the network` |
| Source filter | `IPv4 ranges` |
| Source IPv4 ranges | `0.0.0.0/0` |
| Protocols and ports | `TCP: 80, 443` |

4. Click **"Create"**. The rule activates in less than 30 seconds.

**Option B — gcloud CLI** (requires network admin permissions in the project):

```bash
gcloud compute firewall-rules create allow-disher \
  --allow tcp:80,tcp:443 \
  --source-ranges 0.0.0.0/0 \
  --description "Disher.io HTTP + HTTPS"
```

> **Note:** If you see the error `Request had insufficient authentication scopes`, the VM doesn't have permissions to manage the firewall. Use Option A from the web console.

### AWS (EC2)

1. Go to **EC2 → Instances → select your instance**
2. In the **Security** tab, click the **Security Group**
3. In **Inbound rules**, add:
   - Type: `HTTP`, Port: `80`, Source: `0.0.0.0/0`
   - Type: `HTTPS`, Port: `443`, Source: `0.0.0.0/0`
4. Save the changes.

### Azure (Virtual Machine)

1. Go to your VM in the Azure portal
2. In **Networking**, add **Inbound port rules**:
   - Port: `80`, Protocol: `TCP`, Action: `Allow`
   - Port: `443`, Protocol: `TCP`, Action: `Allow`

## Useful Docker Commands

```bash
# Check service status
sudo docker compose ps

# View all service logs in real time
sudo docker compose logs -f

# Restart a specific service (e.g., backend)
sudo docker compose restart backend

# Stop all services
sudo docker compose down
```
