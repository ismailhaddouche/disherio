# Uninstallation Guide

This document describes the procedures for completely removing the DisherIo platform from your system.

> **Warning**: All operations on this page are irreversible. Run `sudo ./scripts/install.sh backup` before proceeding.

---

## 1. Automated uninstall (recommended)

The universal script handles complete removal with interactive confirmation:

```bash
cd disherio
sudo ./scripts/install.sh uninstall
```

You will be asked to type `SI` to confirm. The script then:

1. Stops and removes project containers (including orphans)
2. Removes the named MongoDB, Redis, uploads, and Caddy volumes
3. Removes locally built project images
4. Deletes generated `.env`, `.credentials`, `Caddyfile`, Compose override,
   MongoDB keyfile, and `config/secrets/`

Encrypted archives under `/var/backups/disherio` are intentionally retained;
remove them separately only after their retention obligation has ended.

---

## 2. Manual uninstall

### Step 1: Stop and remove containers

```bash
cd disherio
docker compose down --volumes --remove-orphans --rmi local
```

This removes the services in the resolved Compose project, its default
networks, named data volumes, and locally built images. Container/image names
are generated by Compose and must not be assumed to use a fixed underscore
format.

### Step 2: Remove named volumes (if not removed by `down --volumes`)

```bash
docker volume rm disherio_mongo_data disherio_redis_data disherio_uploads \
  disherio_caddy_data disherio_caddy_config 2>/dev/null || true
```

### Step 3: Remove generated configuration and secrets

```bash
rm -f .env .credentials Caddyfile docker-compose.override.yml config/mongo-keyfile
rm -rf config/secrets
```

### Step 4: Remove the project directory

```bash
cd ..
rm -rf disherio
```

---

## Summary

| Step | Command | Effect |
|------|---------|--------|
| 1 (auto) | `sudo ./scripts/install.sh uninstall` | Complete removal (containers, volumes, images, config) |
| 1 (manual) | `docker compose down --volumes --remove-orphans` | Remove containers, volumes, orphans |
| 2 | `docker volume rm ...` | Remove named volumes (if needed) |
| 3 | `rm -f ... && rm -rf config/secrets` | Remove generated configuration and secrets |
| 4 | `rm -rf disherio` | Remove project directory |

---

## Soft stop (keep data)

If you only want to stop services temporarily without losing data:

```bash
sudo ./scripts/install.sh stop
# Or:
docker compose stop
```

To restart later:
```bash
sudo ./scripts/install.sh start
```
