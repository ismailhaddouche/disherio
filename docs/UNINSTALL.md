# Uninstallation Guide

This document describes the procedures for completely removing the DisherIo platform from your system.

> **Warning**: All operations on this page are irreversible. Run `sudo ./scripts/install.sh backup` before proceeding.

---

## 1. Automated uninstall (recommended)

The universal script handles complete removal with interactive confirmation:

```bash
cd disherio
sudo ./scripts/install.sh uninstall
```

You will be asked to type `SI` to confirm. The script then:

1. Stops and removes all containers (including orphans)
2. Removes all Docker volumes (MongoDB, Redis, uploads, and Caddy)
3. Removes application images (`disherio_backend`, `disherio_frontend`, `disherio_seed`)
4. Deletes configuration files (`.env`, `.credentials`, `Caddyfile`)

---

## 2. Manual uninstall

### Step 1: Stop and remove containers

```bash
cd disherio
docker compose down --volumes --remove-orphans
```

This removes:
- All DisherIo containers (`disherio_mongo`, `disherio_mongo_init_replica`, `disherio_redis`, `disherio_backend`, `disherio_frontend`, `disherio_seed`, `disherio_caddy`)
- Docker network (`disherio_disherio_net`)
- Persistent volumes (`mongo_data`, `redis_data`, `uploads`, `caddy_data`, `caddy_config`)

### Step 2: Remove named volumes (if not removed by `down --volumes`)

```bash
docker volume rm disherio_mongo_data disherio_redis_data disherio_uploads \
  disherio_caddy_data disherio_caddy_config 2>/dev/null || true
```

### Step 3: Remove custom images

```bash
docker rmi disherio_backend disherio_frontend disherio_seed 2>/dev/null || true
```

> **Note**: Image names are `disherio_backend`, `disherio_frontend`, `disherio_seed` (with underscores, as generated by Docker Compose from the service names).

### Step 4: Remove configuration files

```bash
rm -f .env .credentials Caddyfile
```

### Step 5: Prune unused base images (optional)

```bash
docker image prune -af
```

This removes all dangling and unused images from the system, which may include
the project's `mongo:7`, `redis:7-alpine`, `caddy:2-alpine`, and
`node:24-alpine` base images when no other project references them. DisherIo no
longer installs separate metrics, dashboard, alerting, or exporter images.

### Step 6: Remove the project directory

```bash
cd ..
rm -rf disherio
```

---

## Summary

| Step | Command | Effect |
|------|---------|--------|
| 1 (auto) | `sudo ./scripts/install.sh uninstall` | Complete removal (containers, volumes, images, config) |
| 1 (manual) | `docker compose down --volumes --remove-orphans` | Remove containers, volumes, orphans |
| 2 | `docker volume rm ...` | Remove named volumes (if needed) |
| 3 | `docker rmi disherio_backend disherio_frontend disherio_seed` | Remove custom images |
| 4 | `rm -f .env .credentials Caddyfile` | Remove config files |
| 5 | `docker image prune -af` | Remove unused base images |
| 6 | `rm -rf disherio` | Remove project directory |

---

## Soft stop (keep data)

If you only want to stop services temporarily without losing data:

```bash
sudo ./scripts/install.sh stop
# Or:
docker compose stop
```

To restart later:
```bash
sudo ./scripts/install.sh start
```
